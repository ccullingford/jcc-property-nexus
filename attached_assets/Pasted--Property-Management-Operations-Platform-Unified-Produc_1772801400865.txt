
# Property Management Operations Platform
## Unified Product Specification for Replit Build

This document combines the **architecture, entity model, UI layout, features, and build plan** for creating a property‑management operations platform.

The system combines:

- Shared Outlook inbox
- Task management
- CRM-lite contacts
- Issue / case tracking
- Property & unit context
- RingEX call pop
- Future AI readiness

The goal is **an operations platform where communication creates work**, not just another email client.

---

# Technology Stack (Required)

Replit should build the system using this exact stack.

Frontend
- React
- TypeScript
- Tailwind (preferred)

Backend
- Node.js
- Express or structured API layer

Database
- PostgreSQL

Other
- Microsoft Graph API (Outlook integration)
- RingEX URL call pop integration

Architecture must remain **portable and not dependent on Replit-specific frameworks**.

---

# Core Domain Model

The system revolves around relationships between communication and operational work.

High‑level model:

Property  
└── Unit  
  └── Contact  
    ├── Threads  
    │  └── Messages  
    ├── Calls  
    ├── Issues  
    │  ├── Tasks  
    │  ├── Notes  
    │  └── Emails  
    └── Activity Timeline

Key rule:

Everything must be linkable to:
- Contact
- Property / Unit
- Thread
- Issue
- Task

# Call Pop (RingEX)

Incoming call triggers:

/call-pop?phone=%E

System flow

1 receive phone number  
2 normalize phone number  
3 lookup contact_phones  
4 if match show contact  
5 if not show unknown caller

Call pop screen shows

- contact
- recent activity
- open issues
- quick create issue

# Success Criteria for Version 1

The system is successful when staff can:

1 view shared mailbox threads  
2 claim and assign them  
3 reply to emails  
4 create tasks  
5 add notes  
6 view contact history  
7 receive RingEX call pop with context  

At that point the system already replaces several tools.
