# Part 1 — Contact Import

Add the ability to import contacts in bulk.

## Requirements

Support CSV import with a guided import flow.

### Import flow
1. Upload CSV file
2. Preview rows
3. Map CSV columns to system fields
4. Validate data
5. Show duplicate warnings
6. Import records

### Supported fields
At minimum support:

display_name  
first_name  
last_name  
primary_email  
secondary_email  
primary_phone  
secondary_phone  
contact_type  
notes

### Import behavior
Allow these modes:

- create new only
- update existing by email match
- update existing by phone match
- create or update

### Validation
Validate:

- email format
- phone normalization where possible
- required display name or usable fallback
- duplicate rows in uploaded file

# Part 2 — Contact Matching Improvements

Improve automatic matching between synced messages and existing contacts.

## Matching priority

When syncing or rendering threads, match contacts in this order:

1. exact normalized email match
2. exact normalized phone match if available later
3. manual thread-contact link
4. no match

### Email normalization
Store and compare emails using:
- lowercase
- trimmed values

### Phone normalization
Store and compare phone numbers using:
- E.164 where possible

Do not perform fuzzy matching that may create bad associations.

---

# Part 3 — Unknown Contact Prompt

When a thread has no linked contact and no automatic match is found:

Show a clear prompt in the thread view:

Unknown Contact

Actions:
- Create Contact
- Search Existing Contact
- Dismiss / Ignore for now

### Create Contact flow
Prefill:
- display name from sender
- primary email from sender email

### Search Existing flow
Allow user to:
- search contacts
- select one
- link selected contact to the thread

This must persist to the database.

---

# Part 4 — Duplicate Detection and Cleanup

Add duplicate handling tools.

## Duplicate signals

Flag likely duplicates when:
- same normalized email
- same normalized phone
- same display name plus matching domain or phone pattern

Do not auto-merge duplicates silently.

## Duplicate tools
Provide a review screen or action flow to:
- compare duplicate candidates
- merge duplicate contacts manually
- keep both if needed

### Merge behavior
When merging:
- preserve linked threads
- preserve linked tasks
- preserve notes/history
- preserve all emails and phone numbers when possible
- choose one primary record as survivor
