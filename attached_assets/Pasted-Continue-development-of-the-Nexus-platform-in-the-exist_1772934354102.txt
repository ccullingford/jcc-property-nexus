==================================================
RELEASE / VERSIONING STANDARD (MANDATORY)
==================================================

Nexus must use one single application version at a time.

Adopt Semantic Versioning (SemVer):
MAJOR.MINOR.PATCH

Examples:
- 1.0.0
- 1.1.0
- 1.1.1

Rules:
- MAJOR = breaking or major architectural/platform change
- MINOR = new feature(s) added
- PATCH = bug fixes, minor improvements, small UX corrections

There must never be multiple parallel active versions such as:
- 1.2
- 1.3
- 1.2.1
at the same time in What's New.

Only one current application version exists at a time.

Every future update prompt must:
1. determine the next correct version number
2. increment the application version
3. apply that version consistently to the app and What's New entries
4. summarize why the version increment is MAJOR, MINOR, or PATCH
