# Part 1 — Mailbox Sync Configuration

Add configurable sync settings per mailbox.

## Database changes

Add fields to the existing `mailboxes` table:

sync_history_days (integer default 30)
include_sent_mail (boolean default true)
auto_sync_enabled (boolean default true)
auto_sync_interval_minutes (integer default 5)
last_synced_at (timestamp nullable)

Purpose:

sync_history_days
Controls how far back the initial mailbox sync goes.

Example:
30 = sync last 30 days of mail.

include_sent_mail
Determines whether Sent Items are synced.

auto_sync_enabled
Allows scheduled sync.

auto_sync_interval_minutes
Controls background sync frequency.

# Part 3 — Sync Window (Days)

Mailbox sync must respect:

sync_history_days

Example:

If sync_history_days = 60
only pull messages newer than:

today - 60 days

Implementation:

receivedDateTime >= calculatedDate

Apply this to both inbox and sent mail queries.

# Part 4 — Automatic Sync Scheduler

Add a background sync system.

Backend service:

mailboxSyncScheduler

Behavior:

Every auto_sync_interval_minutes:

Loop through mailboxes where:

auto_sync_enabled = true

Run sync job.

Requirements:

* prevent concurrent sync for the same mailbox
* log sync results
* update last_synced_at

# Part 5 — Automatic Contact Linking

When new messages are synced:

Attempt to match sender/recipient emails to existing contacts.

Steps:

1. Normalize email (lowercase)
2. Search contacts.email table
3. If match found:
   attach contact to thread

If multiple contacts match, prefer primary email.
