Update application version constant to 1.5.5
Add What's New entry explaining the ownership model correction.

------------------------------------------------

PROBLEM

The current schema assumes a contact has a 1:1 relationship with:

association
unit

This is incorrect.

In real property management:

A contact may own MANY units.
Those units may exist in MANY associations.

Example:

Rony Levkovitz
owns 29 units in Shannon Green HOA
owns 9 units in another association

Therefore:

Contact → Unit must be MANY-TO-MANY.

------------------------------------------------

TARGET DATA MODEL

Contacts
Units
Associations

Relationships:

Associations
    1 → many Units

Contacts
    many ↔ many Units

------------------------------------------------

NEW TABLE REQUIRED

Create a relationship table:

contact_units

Columns:

id
contact_id
unit_id
association_id
role
is_primary
created_at
updated_at

------------------------------------------------

ROLE FIELD

Role examples:

Owner
Tenant
Board Member
Vendor
Property Manager

Example:

contact_id: Rony
unit_id: 9531M
association_id: Shannon Green
role: Owner

------------------------------------------------

REMOVE OLD STRUCTURE

If the current system stores:

association_id
unit_id

directly on contacts,

this must be removed or deprecated.

The contact record should NOT store a single unit.

------------------------------------------------

DATA MIGRATION

Create a migration that:

1. Reads the existing contact association/unit fields
2. Creates records in contact_units
3. Preserves the role (Owner if not specified)

Ensure existing relationships remain intact.
